# GT-43 SAC, SEP-41, TTL, and BN254 reconciliation

Primary process 3311 and blind process 3320 independently audited seven cases.
The blind matrix was persisted before any primary/scratchpad read; its delayed
lock was explicitly audited and accepted only after chronology confirmation.

Key corrections:

- Current SAC identity uses `CONTRACT_EXECUTABLE_STELLAR_ASSET`; public
  `name()` helps resolve the asset after SAC identity is established, while
  `admin()` is mutable and is not permanent issuer provenance.
- Current SEP-41 direct transfer accepts `MuxedAddress`; `transfer_from`
  remains Address-based. `live_until_ledger` and `expiration_ledger` describe
  one absolute-ledger allowance boundary.
- Raw JS `Contract.call` is spec-free; generated/static/retained typed clients
  are the stale surfaces after upgrade.
- Current network TTL settings are dated values; extension requires explicit
  threshold/extend-to APIs and P23 restoration is not extension.
- SDK v25 already exposes BN254 wrappers and feature-gated raw Poseidon through
  `Env::crypto_hazmat`; v23 lacks the surface.

Protocol-spec owner candidate: checked-in CAP-0075 says U32Val/P24 in places,
while shipped v25+ `env.json` uses Symbol selectors and the protocol feature
floor is P25. This does not fit the current service-improvements intake and is
preserved here for protocol-owner follow-up.
